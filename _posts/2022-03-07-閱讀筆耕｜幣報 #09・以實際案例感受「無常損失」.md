<p>幣圈一天，人間一年，反覆無常，<strong><u>無常損失（Impermanent Loss）</u></strong>，好詩好詩——本文用真實案例帶大家感受這專有名詞。</p><hr><hr><h2><strong>■ 緣起</strong></h2><p>之前在 <a href="https://matters.news/@penfarming/%E9%96%B1%E8%AE%80%E7%AD%86%E8%80%95-%E5%8E%BB%E4%B8%AD%E5%BF%83%E5%8C%96%E4%BA%A4%E6%98%93%E6%89%80-marble-dao-%E6%96%B0%E9%96%8B%E5%B9%95-%E5%90%AB%E4%B8%89%E5%B9%A3%E7%A9%BA%E6%8A%95%E6%83%85%E5%A0%B1-bafyreiea23vzh6po2rlrip2esavxoxrq6nbgd7xkqatilvbldabe2ddpeq" rel="noopener noreferrer" target="_blank">幣報 #07</a> 領到 20 顆 $RAC（浣熊幣）空投，便把它剖半，放進 JunoSwap 的 <a href="https://junoswap.com/pools/JUNO-RAC" rel="noopener noreferrer" target="_blank">Juno + RAC</a> 流動池裡，這幾突然想到，這池子相對單純，是個實際理解與觀測「無常損失」的好材料。</p><figure class="image"><img src="https://assets.matters.news/embed/c715c055-1a85-45ac-a1f8-74e016078034.png" data-asset-id="c715c055-1a85-45ac-a1f8-74e016078034" referrerpolicy="no-referrer"><figcaption><span>為 JUNO・RAC 池子提供流動性</span></figcaption></figure><hr><h2><strong>■ 資料觀測</strong></h2><p>還記得當初交易完成時的一瞬，那 $RAC 被切成很漂亮的整數——10 顆——所以印象深刻，然 $JUNO 的數額就沒記錄到；之後每每想到，就上去巡一巡，把歷次觀察到的兩幣「<strong>數量變化</strong>」抄寫下來：</p><ul><li>❶ JUNO＿0.431662 + RAC＿11.023077</li><li>❷ JUNO＿0.404109 + RAC＿11.780540</li><li>❸ JUNO＿0.414594 + RAC＿11.485907</li><li>❹ JUNO＿0.394830 + RAC＿12.080336</li><li>❺ JUNO＿0.393864 + RAC＿12.110389</li></ul><figure class="image"><img src="https://assets.matters.news/embed/684721e8-3b1f-47d6-9df3-6a692ca67cf8.png" data-asset-id="684721e8-3b1f-47d6-9df3-6a692ca67cf8" referrerpolicy="no-referrer"><figcaption><span>❺ JUNO 0.393864 + RAC 12.110389</span></figcaption></figure><hr><h2><strong>■ 實際案例：無常損失</strong></h2><p>首先大家要知道，若你把手上的虛擬貨幣拿去質押（Stake），<strong><u>本金的數量固定不變</u></strong>；而換作是「造市」提供流動性（Liquidity Provider, LP）時，因為你的幣放在池裡供用戶交換（Swap），<strong><u>所以本金（兩幣）的數量是此消彼漲</u></strong>，更深入的演示請見 <a href="https://matters.news/@penfarming/%E9%96%B1%E8%AE%80%E7%AD%86%E8%80%95-%E7%84%A1%E5%B8%B8%E6%90%8D%E5%A4%B1%E6%98%AF%E4%BB%80%E9%BA%BC-%E6%8F%90%E4%BE%9B%E6%B5%81%E5%8B%95%E6%80%A7%E4%B9%9F%E8%A6%81%E6%B3%A8%E6%84%8F-bafyreiaiu3whtq4z7tylvslusxfsahietum2mw6xfidtvwbij2gtqpewei" rel="noopener noreferrer" target="_blank">無常損失是什麼？提供流動性也要注意</a> 一文，不過該篇文章數字繁雜，需要一些耐心。</p><p>＊</p><p>回到「無常損失」一詞，這意境太抽象、太詩意。就原文用字 Impermanent Loss 來看：</p><blockquote>傾向譯為「非恆常性損失、非永久性損失」會比較貼切，<strong><u>損失是暫時的，是帳面上的，是未實現的</u></strong>。</blockquote><p>不過，請容許本文繼續追隨這最普遍性的稱呼。</p><blockquote>用最白話的說法，它的意思就是<strong><u>你手上的雙幣「按兵不動 vs 協助造市」兩種情況下的市值差異</u></strong>。</blockquote><p>就實際案例，假設我在觀測期 ❶ 的時候收手，把兩幣取出來，從此單純持有，此謂「按兵不動」也；另一位平行時空的我，則繼續把雙幣浸在池裡，提供流動性，此乃「協助造市」，於是來到觀測期 ❺ 的時候：</p><figure class="image"><img src="https://assets.matters.news/embed/179e9f07-0d95-4744-81ed-2690ea3c6861.png" data-asset-id="179e9f07-0d95-4744-81ed-2690ea3c6861" referrerpolicy="no-referrer"><figcaption><span>以實際案例計算無常損失</span></figcaption></figure><p>匯率取行文當下的時價，於是現今我的 <a href="https://junoswap.com/pools/JUNO-RAC" rel="noopener noreferrer" target="_blank">Juno + RAC</a> 流動池「<strong>帳面上（未實現）</strong>」蒙受 0.11777968 $UST 的無常損失。</p><p>＊</p><p>雖說按兵不動，但實際上 A 可以把雙幣拿去質押，如此一來市值會更高；而協助造市的 B，其市值已經包含 swap fee 的賺取，但並沒有考慮 JunoSwap 平台給予的額外獎勵（目前尚未開通）。<strong><u>若真把所有面向都涵蓋進來，究竟 B 會不會有「損失」還是未定之數</u></strong>。</p><hr><hr><figure class="image"><img src="https://assets.matters.news/embed/75ef2f6a-5d7b-4099-b346-5f6d9e293721.png"
