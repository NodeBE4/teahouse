<p>上週五（4/21）參加了由 Matters 響應 ETH Taipei 所舉辦的 side event「<a href="https://lu.ma/bqbtxkq4" rel="noopener noreferrer" target="_blank">Explore writing and ETH ecosystem in Matters</a>」活動，地點在市政府捷運站附近的 <a href="https://appworks.tw/" rel="noopener noreferrer" target="_blank">AppWorks</a>。</p><p>我記下了一些關於 Matters 文本儲存方式的介紹（是次活動上半場的內容），回到家後查明了相關資料，親身實際操做了一次，整理分享給大家。</p><figure class="image"><img src="https://assets.matters.news/embed/7cc2b88b-d9c4-42a2-9696-2eca267c0b5f.png" data-asset-id="7cc2b88b-d9c4-42a2-9696-2eca267c0b5f" referrerpolicy="no-referrer"><figcaption><span>Explore writing and ETH ecosystem in Matters</span></figcaption></figure><p>👉 <a href="https://liker.land/like1xpkwcv48jqdxym26m8f5wjqa3e7dytq3zr733e?referrer=like1xpkwcv48jqdxym26m8f5wjqa3e7dytq3zr733e" rel="noopener noreferrer" target="_blank">收藏本文的 Writing NFT 版本</a> 👈</p><hr><hr><h2><strong>@Zeck 介紹／回顧 Matters 功能</strong></h2><p>■<strong><u> IPFS 與作品指紋</u></strong></p><p>先是簡介 Matters 站上內容保存方式 IPFS（InterPlanetary File System），透過網路「分散式數據儲存」的方式，在多台電腦上儲存檔案，且與不同使用者分享；為每個檔案賦予獨一無二的亂數值，<strong>可以理解為「指紋」</strong>。</p><blockquote>路徑：打開文章 → 點擊文章上方「分佈式入口」→ 複製「作品指紋」。</blockquote><figure class="image"><img src="https://assets.matters.news/embed/cd773c44-c856-4470-ac99-b8d1eea8e80b.png" data-asset-id="cd773c44-c856-4470-ac99-b8d1eea8e80b" referrerpolicy="no-referrer"><figcaption><span>Matters 上每一篇內容的「作品指紋」</span></figcaption></figure><p>當你想下載某個檔案時，不用再告訴電腦「去哪裡」尋找檔案（這叫做「地址定址」），只需告訴它「要什麼」檔案就可以了（這叫做「內容定址」）<strong>。</strong>詢問誰有包含這個亂數值的檔案，然後 IPFS 網路上的某個人就會將檔案提供給你，<strong>概念上有像是 BitComet、eDonkey 載等 P2P 資源分享軟體，老司機都懂！</strong></p><p>我們可以在 Web3 友好瀏覽器（例如 Brave），把方才的「作品指紋」前面加上「ipfs://」字樣，再貼到網址列。例如若要查找 <a href="https://matters.town/@penfarming/384096-%E6%89%93%E9%80%A0%E7%AC%AC%E4%BA%8C%E5%A4%A7%E8%85%A6-%E8%91%97%E7%9C%BC%E6%96%BC%E8%B3%87%E8%A8%8A%E5%85%A9%E5%A4%A7%E7%89%B9%E6%80%A7-%E8%AE%93%E7%AD%86%E8%A8%98%E7%82%BA-%E8%A1%8C%E5%8B%95-%E5%8B%95%E8%B5%B7%E4%BE%86-bafybeif7dqgxnndypudve4uphyqtj4ecgtloefjwche5yrn2zcpakbpicq" rel="noopener noreferrer" target="_blank">《打造第二大腦》著眼於資訊兩大特性，讓筆記為「行動」動起來</a> 這篇文章，就在網址列輸入：</p><blockquote>ipfs://QmbCfPZ6282okGES4VrXUFdfvF8SQiH8LidQ2rVxszWd9V</blockquote><p>就會得到同一篇「不是」策展在 Matters 上的內容，<strong>實現內容的（多點分散）長久保存，這是相較於中心化伺服器儲存而言</strong>，<a href="https://www.businesstoday.com.tw/article/category/183030/post/202304060074/" rel="noopener noreferrer" target="_blank">有鑒於近期 Xuite 隨意窩關站事件</a>。</p><figure class="image"><img src="https://assets.matters.news/embed/fc4c5baf-28f7-4dbe-9fcb-7273b5de8c07.png" data-asset-id="fc4c5baf-28f7-4dbe-9fcb-7273b5de8c07" referrerpolicy="no-referrer"><figcaption><span>得到同一篇「不是」策展在 Matters 上的內容</span></figcaption></figure><p>順帶一提，因為 IPFS 使用內容定址，一旦某個檔案加入了網路，即擁有不能竄改、不能在網路中移除的特點，<strong>在思想上和​​ </strong><a href="https://chainee.io/what-is-blockchain/" rel="noopener noreferrer" target="_blank"><strong>區塊鏈</strong></a><strong> 很像，但 IPFS 本身和區塊鏈沒有直接關係。</strong></p><p>因此 IPFS 面臨的最大問題是缺乏誘因機制，以保持檔案隨時都在線，而這一點與區塊鏈不同，區塊鏈通常可以透過設計代幣獎勵機制來解決這個問題⋯⋯這裡繼續開展下去，會聊到 Fliecoin（FIL）的種種，先在此打住！</p><hr><hr><p><u>■</u><strong><u> IPNS 與 Planet 閱讀器</u></strong></p><p>根據以上討論，可以得知只要把每一篇文章的「指紋」都給記錄下來，就不怕我的／我追蹤的文章因為 Matters 出了什麼問題（例：關站、或是遭受 DDoS 攻擊）而找不到。<strong>然而，多數人應該不會這麼土法煉鋼，一篇篇手動把這麼多「指紋」另外找個地方備份下來</strong>，這是件苦差事。</p><p>我想像有一個外掛程式，只要輸入某位作者的 ID，就可以打包他在站上所有的內容標題以及對應的「指紋」，技術上應不太難？</p><p>另一個方案是 IPNS（InterPlanetary Name System），<strong>它允許用戶為自己的 IPFS 內容分配一個唯一且可持續的名稱。</strong>換言之，IPNS 提供了一種將 IPFS 內容「易於記憶和分享」的名稱關聯方法。這樣一來，即使內容本身發生變化，用戶也可以使用相同的名稱來訪問它。</p><blockquote>路徑：點擊個人首頁右上角「ᯤ」圖案  → 複製 IPNS 指紋。</blockquote><figure class="image"><img src="https://assets.matters.news/embed/c24b2be4-ec80-4f9d-8435-488f53eb8cc6.png" data-asset-id="c24b2be4-ec80-4f9d-8435-488f53eb8cc6" referrerpolicy="no-referrer"><figcaption><span>讓 IPNS 內容更易於記憶與分享的「IPNS 指紋」</span></figcaption></figure><p>＊</p><p>再次使用 Web3 友好瀏覽器（例如 Brave），只是這次是幫「指紋」前面加上「ipns://」字樣，然後貼到網址列。以下這是 <a class="mention" href="https://matters.news/@penfarming" target="_blank" data-display-name="閱讀筆耕" data-user-name="penfarming" data-id="VXNlcjo0NTE5NA">﻿<span>@閱讀筆耕</span>﻿</a> 的情境：</p><blockquote>ipns://QmbCfPZ6282okGES4VrXUFdfvF8SQiH8LidQ2rVxszWd9V</blockquote><p>很不幸地，我試了許多次，都是 504 Gateway Time-out。</p><figure class="image"><img src="https://assets.matters.news/embed/94332aef-9db7-41ba-82c7-580efcff6ba3.png" data-asset-id="94332aef-9db7-41ba-82c7-580efcff6ba3" referrerpolicy="no-referrer"><figcaption><span>504 Gateway Time-out</span></figcaption></figure><p>＊</p><p>改以站上其他創作者的 IPNS 指紋來嘗試（例如 @米高與小狗），得到一個「不是」策展在 Matters 上的個人頁面，實現一拖拉庫 IPNS 內容策展。</p><figure class="image"><img src="https://assets.matters.news/embed/3811d307-f1cf-4125-971b-083911ee732f.png" data-asset-id="3811d307-f1cf-4125-971b-083911ee732f" referrerpolicy="no-referrer"><figcaption><span>一拖拉庫 IPNS 內容的打包帶走？</span></figcaption></figure><p>＊</p><p>或是你也可以試試 <a href="https://www.planetable.xyz/" rel="noopener noreferrer" target="_blank">PLANET</a>（目前只有 MacOS 版本）這個「類 RSS 軟體」，IPNS 指紋就是你為了要訂閱某個作者而播入的「RSS 種子」，不會錯過許多作者的最新 IPFS 內容，跨平台地追蹤他們。</p><figure class="image"><img src="https://assets.matters.news/embed/98ae8d39-2075-4a98-a81e-53fe2109bfd7.png" data-asset-id="98ae8d39-2075-4a98-a81e-53fe2109bfd7" referrerpolicy="no-referrer"><figcaption><span>不容錯過（跨平台）許多作者的最新 IPFS 內容</span></figcaption></figure><p>不過列表上只會帶出同一位作者「最近 50 篇」的 IPFS 內容，而且有一些內容無法妥善地視覺化呈現出來。</p><p>此外，在 <a href="https://www.planetable.xyz/" rel="noopener noreferrer" target="_blank">PLANET</a> 裡面沒有辦法使用 command+F（ctrl+F）功能，無法以關鍵字反查到特定內容，有點難受啊！</p><figure
