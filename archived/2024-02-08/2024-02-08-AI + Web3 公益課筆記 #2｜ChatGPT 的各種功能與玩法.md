referrerpolicy="no-referrer"><figcaption>AI + Web3 公益課</figcaption></figure><hr><h2>▇  <strong>人物訪談（嘉賓：孟岩）</strong></h2><h3>▍產業心路歷程</h3><ul><li><p>代幣標準 ERC-3525 設計者。</p></li></ul><blockquote><p>💡外部資料補充⚡️<br class="smart">ERC-3525 是一個新的以太坊通證標準，由 Solv Protocol 團隊提出和設計⋯⋯定義了一個新的資產類別——半同質化代幣（SFT），即 Semi-Fungible Token，顧名思義，是介於 FT和 NFT 之間。<br class="smart">⠀<br class="smart">FT 和 NFT 已經可以用來標記現實生活中的一切資產，雖然確實如此，但效率太低，因為有太多的資產「有差別，但差別不大」⋯⋯這是 SFT 想要解決的問題。[1]</p></blockquote><ul><li><p>2015 年，在 IBM 商業戰略改變的情境下，開始學習區塊鏈上的知識。</p></li></ul><ul><li><p>2017 年，從 IBM 離職，專精投入區塊鏈領域。</p><ul><li><p><strong>教育步道培訓</strong><br class="smart">提出 Token Economic（通證經濟）的一些原則。</p></li></ul><ul><li><p><strong>DeFi 賽道 Solv Protocol</strong><br class="smart">市場上缺少一種適合表達「票據」的資產，一開始定位為金融 NFT，最終探索出來為半同質化代幣（SFT）的升級版。</p></li></ul><ul><li><p><strong>RWA</strong><br class="smart">與三個國家的央行、國際組織（世界銀行）建立合作關係，開發一些服務實體經濟領域的應用。</p></li></ul></li></ul><blockquote><p>💡外部資料補充⚡️<br class="smart">Token 翻譯成「通證」會比「代幣」來的理想。</p><p>⠀<br class="smart">通證是覆蓋比較大的超集合，除了轉帳，還多了質押（stake）、委託（delegate）、燒毀（burn）等用途。<br class="smart">⠀<br class="smart">代幣是覆蓋比較小的子集合，只有轉帳的概念。<br class="smart">⠀<br class="smart">社群通證的金錢性固然比不上反映一般商品價值而且歷史悠久的法定貨幣，但反過來也擁有諸多過往無法想像的優勢，<strong>最明顯的是精準地反映和承載社群所重視的價值，把抽象的價值化為具體的行動清單（actionable items）</strong>，讓體現核心價值不只流於遙不可及的願景，產出階段性成果，過程變得比較有趣 [2]。</p></blockquote><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/6a9af4c5-c0d5-4f1b-9f5d-f3fe5c24f71b.png/public" referrerpolicy="no-referrer"><figcaption>圖片來源：高重建《區塊鏈社會學》</figcaption></figure><p>　</p><h3>▍怎麼樣看待區塊鏈產業週期？</h3><ul><li><p>「<strong>Web3 支付／穩定幣支付</strong>」能否站穩腳步，落地應用於實體經濟，是區塊鏈產業爆發的關鍵⋯⋯希望這一次邁入牛市後，能掙脫「進三步退兩步」的四年興衰週期，走一個比較長期的上升趨勢。</p></li><li><p>「<strong>Web3 支付／穩定幣支付</strong>」的特性</p><ul><li><p>跨越 Web2 信任邊界，並解決其高費用、低效用等問題。</p></li></ul><ul><li><p>不僅僅是只有「轉帳」，還能帶有「交易」性質——把我們早習以為常的收執聯、發票、先買後付、分期支付、團購等應用場景都帶進 Web3 裡。</p></li></ul><ul><li><p>終身不丟交易紀錄，儲存在區塊鏈上面⋯⋯像這樣的新型態應用，會像電子郵件般，逐漸滲入我們的生活。</p></li></ul><ul><li><p>在非洲，已經被積極地採用。</p></li></ul></li></ul><p>　　</p><h3>▍看好什麼賽道？</h3><ul><li><p>看好 ZK（零知識證明）技術與發展。ZK 可以讓很多原本只能在區塊鏈上做的事情，被解放出來。</p><ul><li><p>有望靠著 ZK 解決隱私保護與監管的問題。</p></li></ul><ul><li><p>這是一個把「1000 萬億美元」級別的資金體量，搬運到鏈上的大工程。</p></li></ul></li></ul><p>　　</p><h3>▍給 Web3 開發者／投資人的建議</h3><ol><li><p>個人短期主義（賭性）vs 長期構建，兩者必須區分清楚。</p></li><li><p>人窮還得多讀書，學習一些基礎知識（例：經濟學、英文），長期必能帶來回報與價值。</p></li><li><p>敬畏「原則」，要相信你所做的每一件事情是有後果（代價）的。</p></li></ol><hr><h2>▇  主題大綱（講師：陳財貓）</h2><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/63428e2a-71df-4182-b097-5eb0dac3fe4d.png/public" referrerpolicy="no-referrer"><figcaption>主題大綱</figcaption></figure><hr><h2>▇  主題：ChatGPT 的各種功能與玩法</h2><h3>▍<strong>ChatGPT 的內建功能</strong></h3><ul><li><p>三種內建功能（Code Interpreter、Web Browsing、Dall-E）原本是分開的，後來合併了。</p></li><li><p>Code Interpreter 與我們的生活最息息相關。</p></li><li><p>OpenAI 聯合創辦人 <a target="_blank" rel="noopener noreferrer nofollow" href="https://www.bilibili.com/video/BV1MP411X7g8/?share_source=copy_web&t=657">Greg Brockman TED 演說 10:58</a> 示範了 Code Interpreter 具備讀取檔案、自動撰寫 Python 程式的能力，因此可以讓它幫你分析資料、製作報表、修改圖片、製作幻燈片、清理資料集等。</p></li><li><p>Code Interpreter 運作原理像是一個「沙盒」，為其中搭載 200 多個軟件包的執行，提供了一個隔離環境，產出成果。</p></li></ul><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/f58a62d0-e466-4f6f-8270-e3032bddadf3.png/public" referrerpolicy="no-referrer"><figcaption></figcaption></figure><ul><li><p>Web Browsing 上網功能對於緩解「<a target="_blank" rel="noopener noreferrer nofollow" href="https://matters.town/@penfarming/514124-ai-web3-%E5%85%AC%E7%9B%8A%E8%AA%B2%E7%AD%86%E8%A8%98-1-%E5%A4%A7%E5%9E%8B%E8%AA%9E%E8%A8%80%E6%A8%A1%E5%9E%8B%E6%98%AF%E4%BB%80%E9%BA%BC-bafybeia32tconosnjoh2p7g77fai47zfbsmm6mikfqkjw7ighu7strlfuq">幻覺（hallucination）</a>」問題是有幫助的。</p></li><li><p>Dall-E 專門用於生成 AI 圖像，以填補 ChatGPT 中只能輸入及生成文字的限制，在第四堂課做更深入介紹。</p></li></ul><hr><h3>▍<strong>GPTs</strong></h3><ul><li><p>GPTs 是被封裝來為單一場景服務的 Agent，稍後會進一步解說 Agent 的概念。</p></li><li><p>對話式的 GPTs 創建流程，不用寫程式碼。</p></li><li><p>GPTs 的三個元素：</p><ul><li><p><strong>Instructions</strong><br class="smart">給 GPTs 的指令／指南，指導模型「完成特定任務或遵循特定流程」的規則。</p></li><li><p><strong>Actions</strong><br class="smart">GPTs 執行的動作或操作，能與外部 API 的串接，例如標記其他 GPTs 協同作用。</p></li><li><p><strong>Knowledge</strong><br class="smart">GPTs 擁有或可造訪的資料庫。</p></li></ul></li></ul><hr><h3>▍<strong>Agent（智能體）</strong></h3><ul><li><p>Agent 翻譯成「智能體」會比「代理人」來的理想。</p></li></ul><blockquote><p>💡外部資料補充⚡️<br class="smart">Agent一詞源於拉丁語的「Agere」，意為「to do」。在大語言模型（LLM）的語境裡，Agent 指的是具備自主理解、規劃決策和執行複雜任務的智能體。[3]</p></blockquote><ul><li><p>比喻：人在道路上開車，看到貓突然竄出，於是踩了煞車。</p><ul><li><p>人 → 智能體</p></li><li><p>世界 → 環境（道路）</p></li><li><p>眼睛 → 傳感器</p></li></ul><ul><li><p>腳 → 執行器（讓人可以作用於這個世界）</p></li></ul></li></ul><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/7a8f9d91-0bba-40db-9d64-64ce831edf42.png/public" referrerpolicy="no-referrer"><figcaption>Agent 是什麼</figcaption></figure><ul><li><p>Agent 的組成：</p><ul><li><p><strong>Planing（規劃）</strong><br class="smart">將大任務分解為更小、更易管理的子目標。<br class="smart">能從過去的行動自我批評和反思，對未來的步驟進行改良。</p></li><li><p><strong>Memory（記憶）</strong><br class="smart">包括短期記憶和長期記憶，後者通常利用 <a target="_blank" rel="noopener noreferrer nofollow" href="https://aws.amazon.com/tw/what-is/vector-databases/">外部向量儲存</a> 和快速檢索來實現。</p></li><li><p><strong>Tools（工具）</strong><br class="smart">智能體能使用的資源，以及學會調用外部 API 的能力，為了獲取模型權重中不足的額外資料。</p></li><li><p><strong>Action （行動）</strong></p></li></ul></li></ul><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/f0889dda-d50d-421d-9144-e69b4a5020fc.png/public" referrerpolicy="no-referrer"><figcaption>Agent 的組成</figcaption></figure><hr><h3>▍<strong>RAG（檢索增強生成）</strong></h3><ul><li><p>RAG（Retrieval-Augmented Generation）</p></li></ul><blockquote><p>💡外部資料補充⚡️<br class="smart">法官與法庭助理的比喻 [4]。<br class="smart">⠀<br class="smart">當某個案件特別硬核，需要特殊專業知識時，法官會派遣法庭助理到法律圖書館，尋找可以引用的先例和特定案例。<br class="smart">⠀</p><p>同理，大語言模型就像是一位法官，可以「判決」我們的各種呈堂文本。但為了提供引用來源的權威答案，該模型需要助手——RAG——進行若干研究。</p></blockquote><ul><li><p>大語言模型受到其預訓練（Pre-trained）數據的限制，缺乏對最新事件的了解，透過 RAG 從外部資料庫來源「<strong>檢索」</strong>最新文檔，然後將它們作為上下文訊息，輸入到一個「<strong>生成</strong>」模型裡（例：Transforme<a target="_blank" rel="noopener noreferrer nofollow" href="https://matters.town/@penfarming/514124-ai-web3-%E5%85%AC%E7%9B%8A%E8%AA%B2%E7%AD%86%E8%A8%98-1-%E5%A4%A7%E5%9E%8B%E8%AA%9E%E8%A8%80%E6%A8%A1%E5%9E%8B%E6%98%AF%E4%BB%80%E9%BA%BC-bafybeia32tconosnjoh2p7g77fai47zfbsmm6mikfqkjw7ighu7strlfuq">r</a> 模型），以求更準確、豐富地響應我們的要求。</p></li><li><p>對於緩解「<a target="_blank" rel="noopener noreferrer nofollow" href="https://matters.town/@penfarming/514124-ai-web3-%E5%85%AC%E7%9B%8A%E8%AA%B2%E7%AD%86%E8%A8%98-1-%E5%A4%A7%E5%9E%8B%E8%AA%9E%E8%A8%80%E6%A8%A1%E5%9E%8B%E6%98%AF%E4%BB%80%E9%BA%BC-bafybeia32tconosnjoh2p7g77fai47zfbsmm6mikfqkjw7ighu7strlfuq">有限的上下文窗口（Context Window）</a>」、「<a target="_blank" rel="noopener noreferrer nofollow" href="https://matters.town/@penfarming/514124-ai-web3-%E5%85%AC%E7%9B%8A%E8%AA%B2%E7%AD%86%E8%A8%98-1-%E5%A4%A7%E5%9E%8B%E8%AA%9E%E8%A8%80%E6%A8%A1%E5%9E%8B%E6%98%AF%E4%BB%80%E9%BA%BC-bafybeia32tconosnjoh2p7g77fai47zfbsmm6mikfqkjw7ighu7strlfuq">幻覺（hallucination）</a>」問題皆有幫助。</p></li></ul><hr><h3>▍Embedding 是什麼？</h3><ul><li><p><strong>將單詞、句子或文檔（壓縮後）映射為一個固定大小的向量表示</strong>。</p></li><li><p>這些向量表示能夠捕捉到詞句的語義和語境，使得計算機可以處理自然語言數據。</p></li><li><p>比喻：Openbook 開卷考試要讓 GPT 偷看的一張又一張小紙條（因為 book 的上下文不夠用，所以打小抄，需要用到時把小抄抽出來，方便隨時看）。</p></li></ul><figure class="image"><img src="https://imagedelivery.net/kDRCweMmqLnTPNlbum-pYA/prod/embed/67d3b9e6-e85f-4044-90c1-414f07294c20.png/public"
